# Spec: handoff
> Change: sdr-conversacional | Phase: sdd-spec | Date: 2026-04-23

## Overview

Defines the handoff protocol: when to trigger it, the deal brief JSONB structure, the founder notification format, the draft approval gate, and what happens after founder approval or override. Handoff is the final action of a successful SDR conversation.

---

## Requirements

### REQ-hand-001: Handoff trigger conditions
**Priority**: MUST
**Rule**: DA-SDR-03

Handoff is triggered when ANY of these conditions is met:
1. `score_total >= 65` (primary trigger)
2. Prospect explicitly asks to speak with a human ("quiero hablar con alguien", "¿puedo hablar con el equipo?")
3. Prospect asks about pricing in a way that indicates readiness to buy ("¿cuánto cuesta para mis fincas?")

On handoff trigger, the SDR MUST:
1. Generate the deal brief (via SP-SDR-06)
2. Save deal brief to `sdr_prospectos.deal_brief`
3. Update `sdr_prospectos.status = 'qualified'`
4. Send founder notification via WhatsApp
5. Continue conversation with the prospect while awaiting founder approval (auto-responses only)

**Acceptance criteria**:
- [ ] `score_total >= 65` is checked after every score update — triggers handoff in the same turn
- [ ] Explicit human request is detected via keywords: "hablar con alguien", "hablar con ustedes", "quiero hablar con el equipo", "persona real"
- [ ] Price readiness is detected via: "¿cuánto cuesta?", "¿qué precio tienen?", "¿cuánto es?" — after turn 3
- [ ] Once handoff is triggered, it cannot be un-triggered in the same session

### REQ-hand-002: Deal brief JSONB schema
**Priority**: MUST
**Rule**: DA-SDR-07

The deal brief MUST contain exactly these fields:

```json
{
  "nombre_contacto": "string | null",
  "empresa": "string | null",
  "cargo": "string | null",
  "segmento_icp": "exportadora | ong | gerente_finca | otro",
  "narrativa_asignada": "A | B",
  "qualification_score": 0,
  "scores_por_dimension": {
    "eudr_urgency": 0,
    "tamano_cartera": 0,
    "calidad_dato": 0,
    "champion": 0,
    "timeline_decision": 0,
    "presupuesto": 0
  },
  "fincas_en_cartera": null,
  "cultivo_principal": "string | null",
  "pais": "string | null",
  "eudr_urgency_nivel": "alta | media | baja | ninguna | desconocida",
  "sistema_actual": "string | null",
  "objeciones_manejadas": [],
  "punto_de_dolor_principal": "string | null",
  "compromiso_logrado": "reunion | piloto | ninguno",
  "fecha_propuesta_reunion": "ISO8601 string | null",
  "conversacion_resumen": "string (max 200 chars)",
  "turns_total": 0,
  "questions_asked": 0,
  "handoff_trigger": "score_threshold | human_request | price_readiness"
}
```

**Acceptance criteria**:
- [ ] All fields are present in every deal brief — null is valid for optional fields, absent is not
- [ ] `qualification_score` matches `score_total` from `sdr_prospectos` at handoff time
- [ ] `conversacion_resumen` is a single-paragraph summary of the conversation, max 200 characters
- [ ] `handoff_trigger` records which condition triggered the handoff
- [ ] Deal brief is generated by SP-SDR-06 in a separate LLM call (not embedded in the main turn response)

### REQ-hand-003: Founder notification format
**Priority**: MUST
**Rule**: DA-SDR-03 (human-in-the-loop for pilot proposals)

The founder notification MUST follow this format, sent to the configured `FOUNDER_PHONE` environment variable:

```
⚡ LEAD CALIFICADO — Wasagro SDR

Score: {score_total}/100 | {segmento_icp} | {narrativa_asignada}

👤 {nombre_contacto} — {cargo}
🏢 {empresa}
📍 {pais} | {cultivo_principal}
🌾 {fincas_en_cartera} fincas
📋 Sistema actual: {sistema_actual}
🔥 EUDR: {eudr_urgency_nivel}
💰 Presupuesto: {presupuesto score}/10

Dolor principal: {punto_de_dolor_principal}
Objeciones manejadas: {objeciones_manejadas.join(', ') | 'ninguna'}

─────────────────────────────
BORRADOR DE PROPUESTA:

{draft_pilot_proposal_message}

─────────────────────────────
Responde *SÍ* para enviar al prospecto.
Responde *NO* para descartar.
Escribe una corrección para ajustar y enviar.
```

**Acceptance criteria**:
- [ ] `FOUNDER_PHONE` is read from env vars — NOT hardcoded
- [ ] Notification is sent to `FOUNDER_PHONE` via the same Evolution/Meta sender used for prospects
- [ ] Notification includes ALL fields from the deal brief in the formatted template above
- [ ] `draft_pilot_proposal_message` is the actual message that would be sent to the prospect — max 300 characters
- [ ] `sdr_prospectos.founder_notified_at` is set when notification is sent

### REQ-hand-004: Draft approval gate
**Priority**: MUST
**Rule**: DA-SDR-03, P7 (no irreversible action without approval)

After sending the founder notification, the SDR MUST wait for founder response. The pipeline handles founder messages with `FOUNDER_PHONE` as sender:

| Founder response | Action |
|-----------------|--------|
| "SÍ" (case-insensitive) | Send draft to prospect, update `status = 'piloto_propuesto'` |
| "NO" (case-insensitive) | Close session, update `status = 'descartado'`, no message to prospect |
| Any other text | Treat as override message — send that text to prospect instead of draft |

**Acceptance criteria**:
- [ ] While awaiting founder approval, ANY message from the prospect gets an auto-response: "Un momento — estoy preparando la propuesta para tu situación específica."
- [ ] The auto-response is sent max once per 10 minutes if prospect sends multiple messages
- [ ] Founder "SÍ" within 24 hours sends the draft; after 24 hours the session stays open but status changes to `piloto_propuesto_pendiente`
- [ ] Founder "NO" is logged in LangFuse as event `sdr_founder_rejected` with `{score, reason: 'manual_reject'}`
- [ ] Founder override text is logged as `sdr_founder_override` — the original draft is archived in `sdr_interacciones`

### REQ-hand-005: Draft pilot proposal content
**Priority**: MUST
**Rule**: SDR-G2 (no H1/H2 promises), SDR-G5 (pilot needs human approval)

The draft pilot proposal message MUST include:
1. A concrete starting point (e.g. "piloto de 4 semanas con 3 fincas")
2. A specific next step (meeting call, WhatsApp demo)
3. A commitment request ("¿podemos agendar 20 minutos?")
4. NO pricing unless founder approves it in override

The message MUST NOT include:
- Features not yet built (H1/H2 roadmap items)
- Specific pricing without founder review
- Hard commitments on timelines ("en 48 horas tienes X")
- Competitor comparisons

**Acceptance criteria**:
- [ ] Draft is generated by SP-SDR-06 using deal brief as input
- [ ] Draft length: 150–300 characters (2–3 sentences for WhatsApp)
- [ ] Draft ends with a question or call to action
- [ ] If `compromiso_logrado = 'reunion'` in deal brief, draft includes meeting scheduling link or request

---

### REQ-hand-006: Post-approval prospect message and follow-up
**Priority**: MUST

After founder approves and the pilot proposal message is sent:
1. `sdr_prospectos.status = 'piloto_propuesto'`
2. `sdr_prospectos.deal_brief.compromiso_logrado = 'piloto'` is updated
3. LangFuse event `sdr_pilot_proposed` fires
4. If prospect responds positively to the proposal, the SDR auto-responds with calendar link (if `DEMO_BOOKING_URL` is set) or asks for preferred time
5. If `DEMO_BOOKING_URL` is set, the SDR sends: "Perfecto — puedes agendar aquí: {DEMO_BOOKING_URL}"
6. If not set, the SDR says: "Perfecto — ¿cuándo tienes 20 minutos disponibles?"

**Acceptance criteria**:
- [ ] `DEMO_BOOKING_URL` is used if available (falls back to manual scheduling)
- [ ] `reunion_agendada_at` is updated when prospect confirms a time
- [ ] `status` transitions: 'piloto_propuesto' → 'reunion_agendada' when meeting is confirmed
- [ ] LangFuse event `sdr_meeting_scheduled` fires when meeting is confirmed

## Scenarios

### SC-hand-01: Score reaches 65 — deal brief generated in same turn

**Given**: Turn 6. Score goes from 58 → 68 after eudr_urgency confirmed.

**When**: Score update is processed.

**Then**: In the SAME turn (6): deal brief generated, `status = 'qualified'`, founder notified, prospect receives: "Interesante lo que me cuentas sobre la presión de tu comprador europeo. Un momento — estoy preparando algo específico para tu operación."

---

### SC-hand-02: Founder approves — pilot proposal sent

**Given**: Founder notified. Founder sends "SÍ" 5 minutes later.

**When**: Pipeline processes `FOUNDER_PHONE` message "SÍ".

**Then**: Draft pilot proposal sent to prospect. `status = 'piloto_propuesto'`. LangFuse event `sdr_pilot_proposed`. Founder receives confirmation: "✅ Propuesta enviada a {nombre_contacto}."

---

### SC-hand-03: Founder overrides draft

**Given**: Founder notified with draft. Founder replies: "Manejamos 5 fincas gratis el primer mes para exportadoras — pruébalas."

**When**: Pipeline receives founder override (non-SÍ/NO text from FOUNDER_PHONE).

**Then**: Founder's text is sent to prospect instead of the draft. `sdr_interacciones` records `tipo = 'founder_override'`. LangFuse event `sdr_founder_override`. `status = 'piloto_propuesto'`.

---

### SC-hand-04: Prospect asks to speak with human before score threshold

**Given**: score_total = 45. Prospect says "prefiero hablar directamente con alguien de Wasagro".

**When**: Pipeline detects human_request handoff trigger.

**Then**: Handoff triggers regardless of score. Deal brief generated with `handoff_trigger = 'human_request'`. Founder notified. Prospect receives: "Perfecto — le aviso a alguien del equipo para que te contacte directamente. ¿Cuál es el mejor horario para ti?"
